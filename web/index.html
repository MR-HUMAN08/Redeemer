REDEEMER
Redis Enumeration Challenge
DIFFICULTY: VERY EASY TYPE: ENUMERATION SERVICE: REDIS
Mission Brief

You've been tasked with enumerating a target network. Intelligence suggests that a Redis database service is exposed somewhere on the network. Your mission: locate the service, enumerate its contents, and retrieve sensitive data.

Target Service
Redis
Estimated Time
15-30 min
Skills Required
Basic Linux
Learning Objectives
  • Perform network reconnaissance using Nmap
  • Identify and enumerate Redis services
  • Connect to Redis using redis-cli
  • Execute Redis commands for information gathering
  • Identify security misconfigurations
  • Extract sensitive data from databases
Tools Available
  • dig - DNS lookup and query tool
  • nslookup - DNS query and hostname resolution tool
  • nmap - Network scanning and service detection
  • redis-cli - Redis command-line client
  • netcat - Network connectivity testing
  • curl - HTTP requests and API testing
  • Full Linux terminal - Complete command-line environment
Instructions
  • dig/nslookup: Use these DNS lookup tools to find the IP address of the target using DNS enumeration. This helps identify target systems on the network.
  • Nmap: Use Nmap for network reconnaissance to scan the IP range 172.20.0.0/16 and discover all active Redis servers. Scan the target IP along with its ports to determine the network status.
  • Port Discovery: Identify the Redis server port using internet research or your browser to find the default Redis port.
  • Redis-CLI: Use redis-cli to connect to the Redis command-line interface and access the stored information.
  • Flag Retrieval: Search for the flag inside the Redis database and submit it to complete the challenge.
OS Instance

Launch your attack machine to begin the enumeration challenge. The OS instance includes all necessary tools and connects directly to the target network.

This is an intentionally vulnerable lab environment for educational purposes only. Do not use these techniques on systems you do not own or have explicit permission to test.
